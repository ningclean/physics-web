# 如何与 AI 编程助手高效沟通

为了让我们未来的沟通更加高效，特别是面对更复杂的逻辑或需求时，您可以参考以下几个维度的建议，这能帮助我更精准地理解您的意图。

## 1. 通用提问模板：现状 + 动作 + 期望

这是最万能的沟通结构：

*   **现状 (Context)**: “目前在 `ProjectileMotionScene.js` 中……” 或 “现在的首页布局是……”
*   **动作/问题 (Action/Problem)**: “当我点击 X 时……” 或 “我想增加一个 Y 功能……”
*   **期望 (Expectation)**: “我希望看到 Z 效果……” 或 “数据应该显示为……”

## 2. 针对不同类型的任务

### A. 修复 Bug
*   **关键点**：**复现步骤**。
*   **示例**：
    *   *不够好*：“小球不动了。”
    *   *很好*：“在‘平抛运动’场景中，当我把‘初速度’设为 0 时，小球卡在原地，控制台没有报错。我希望它能垂直下落。”

### B. 开发新功能
*   **关键点**：**交互流程**和**数据流向**。
*   **示例**：
    *   *不够好*：“加一个重置按钮。”
    *   *很好*：“在控制面板下方加一个‘重置’按钮。点击后，不仅要让小球回到原点，还要把所有滑动条（如速度、角度）都恢复到默认值，并且清空图表数据。”

### C. 修改 UI/样式
*   **关键点**：**参照物**和**布局术语**。
*   **示例**：
    *   *不够好*：“把那个字弄大点，往右边移。”
    *   *很好*：“首页的卡片标题（h3）字号调大到 1.2rem。卡片内部的文字内容希望左对齐，但整个卡片网格（Grid）希望在页面居中显示。”

## 3. 一些“暗号” (专业术语)

如果您能使用一些准确的术语，我会理解得更快：

*   **组件 (Component)**: 指页面上的某个独立模块（如 `ControlPanel`, `RealTimeChart`）。
*   **状态 (State)**: 指随时间变化的数据（如 `this.phys`, `this.params`）。
*   **回调 (Callback)**: 指某个事件发生后执行的函数（如 `onUpdate`, `onChange`）。
*   **路由 (Route)**: 指页面跳转逻辑（如 `#scene=pendulum`）。

## 4. 举个例子

假设您想给“平抛运动”增加一个显示“最大高度”的功能：

*   **普通沟通**: “我想在平抛运动里看它最高能飞多高。”
    *   *(我会做，但我可能会随便找个地方显示)*
*   **高效沟通**: “我想在 `ProjectileMotionScene` 的**控制面板**（或者**公式显示区**）增加一项数据显示。当小球运动时，实时计算并显示当前的**最大高度 (Max Height)**。如果小球落地了，这个数值应该保持在它达到的最高点。”
    *   *(这样我就知道确切的 UI 位置、计算逻辑和更新时机)*

## 总结

如果遇到复杂需求，只要多问自己一句：“**AI 知道我要改哪个文件、在什么时机触发、最终长什么样吗？**” 如果答案是肯定的，那就是一个完美的指令！
